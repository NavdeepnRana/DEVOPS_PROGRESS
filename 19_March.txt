Core Concepts (First Principles)

Docker: A tool used to run applications inside isolated environments called containers
Image: A pre-built template used to create containers
Container: A running instance created from an image
Volume: Storage that keeps data safe even if the container is removed
Network: Helps containers communicate with each other

Container Commands

Run Container
docker run -d --name <name> <image>
Starts a container in the background with a given name

List Containers
docker ps → shows running containers
docker ps -a → shows all containers (running + stopped)

Stop Container
docker stop <container>
Stops a running container

Remove Container
docker rm <container>
Deletes a stopped container

docker rm -f <container>
Force deletes a container even if it is running

Debugging Commands

View Logs
docker logs <container>
Shows output generated by the container

docker logs -f <container>
Shows live logs continuously

Inspect Container
docker inspect <container>
Displays detailed configuration and metadata

Resource Usage
docker stats
Shows CPU, memory, and usage details of containers

Running Processes
docker top <container>
Displays processes running inside the container

Enter Container
docker exec -it <container> bash
Opens terminal access inside the container

Volumes

Create Volume
docker volume create <name>
Creates storage for persistent data

List Volumes
docker volume ls
Shows all volumes

Inspect Volume
docker volume inspect <name>
Displays detailed info about a volume

Use Volume
docker run -v <volume>:<path> <image>
Attaches volume to a container

Mount Path (host side)
/var/lib/docker/volumes/<volume>/_data
Location where volume data is stored in the system

Networks

List Networks
docker network ls
Shows available networks

Create Network
docker network create <name>
Creates a new network

Inspect Network
docker network inspect <name>
Shows network details

Remove Network
docker network rm <name>
Deletes a network

Bulk Commands

Stop all containers
docker stop $(docker ps -q)

Remove all containers
docker rm $(docker ps -aq)

Force remove all containers
docker rm -f $(docker ps -aq)

Remove all images
docker rmi $(docker images -q)

Important Differences

logs → used to check container output
inspect → used to see configuration and internal details
stats → used to monitor performance and resource usage
top → used to view running processes inside container

Network Types

bridge → default network used by Docker
host → container shares host network (no isolation)
none → container has no network access

Apache Deployment Task

docker pull httpd
Downloads Apache image

docker images
Lists available images

docker run -d --name apache-web -p 8081:80 httpd
Runs Apache container and maps port

docker ps
Checks if container is running

Access in browser:
http://localhost:8081

Key Mental Models

Container → temporary execution environment
Volume → permanent data storage
Network → connection between containers

Common Mistakes

Confusing container ID with image ID
Forgetting to use -d (runs container in foreground)
Wrong port mapping format
Trying to remove a container that is still running

Debugging Flow

Enter the container
Check logs
Inspect configuration
Check running processes
