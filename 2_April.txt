Docker Commands – Practice Notes
Mental Model

Container = small isolated environment
Image = base blueprint
Environment variable = configuration/settings
Logs = output generated by container

Q1: Run nginx + env

docker run -d -e ENV_MODE=production --name my_nginx nginx

-d → runs container in background
-e → sets environment variable
--name → assigns container name

Q2: Logs

docker logs my_app
Used to view container output

docker logs -f my_app
Used to see logs continuously (live output)

Q3: Start / Stop

docker start web_server
Starts an existing container

docker stop web_server
Stops a running container

Mental Understanding

run → creates a new container and starts it
start → starts an already created container
stop → safely stops the container
logs → used to check what is happening inside

Common Mistakes

Forgetting to pass -e when environment variables are needed
Using logs without -f when expecting live output
Confusing docker run with docker start

Practice

docker run -d -e MODE=test nginx
docker logs -f <container_id>
docker stop <container_id>
docker start <container_id>
